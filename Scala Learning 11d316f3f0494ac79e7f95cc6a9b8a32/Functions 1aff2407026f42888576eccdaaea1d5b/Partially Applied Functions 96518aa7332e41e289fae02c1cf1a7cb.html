Partially Applied Functions

Partially Applied Functions in Scala

Partially applied functions are a feature in Scala that allows you to "fix" a certain number of arguments for a function, producing a new function with fewer arguments. By doing this, you essentially "pre-apply" some of the arguments, leaving the rest to be filled in later.

Basic Concept

Consider a function that takes three arguments. If you apply just one or two of the arguments, you get a function back that waits for the remaining arguments. This new function is a partially applied function.

Examples

1. Basic Partial Application

Let's start with a simple function that adds three numbers:

def addThreeNumbers(a: Int, b: Int, c: Int): Int = a + b + c

Now, we can partially apply this function:

val addWithFiveAndSix = addThreeNumbers(5, 6, _: Int)
println(addWithFiveAndSix(7)) // Output: 18 (because 5 + 6 + 7 = 18)

In the example above, addWithFiveAndSix is a new function that expects just one argument because the first two arguments are already provided.

2. Using Placeholder _ Multiple Times

You can use the _ placeholder multiple times to represent multiple parameters:

val addWithFive = addThreeNumbers(5, _: Int, _: Int)
println(addWithFive(6, 7)) // Output: 18

3. Partially Applying Without Placeholder

You can also create a partially applied function without the _ placeholder by using an underscore after the function name:

val addOnlyTwo = addThreeNumbers(5, _: Int, _: Int)
val resultFunction = addThreeNumbers _
println(resultFunction(1, 2, 3)) // Output: 6

4. Partially Applied Functions in Higher-Order Functions

Partially applied functions are often used in higher-order functions:

def processList(numbers: List[Int], operation: Int => Int): List[Int] = numbers.map(operation)

val numbers = List(1, 2, 3, 4)
val result = processList(numbers, (_: Int) * 2)
println(result) // Output: List(2, 4, 6, 8)

In the example above, the multiplication operation is partially applied, and the actual number from the list is filled in during the map operation.

Conclusion

Partially applied functions are a powerful feature in Scala that allows you to transform functions and pre-apply some of their arguments. They offer a concise way to generate new functions based on existing ones and are especially useful in functional programming patterns and higher-order functions.
