Variables and Types

Variables and types are foundational concepts in Scala, and they play a crucial role in defining how data is stored and manipulated. In Scala 2.12, we have two main ways to declare variables: using val for immutable variables and var for mutable ones. Let's dive into these concepts.

Immutable Variables (val)

Immutable variables are declared using the val keyword. Once assigned, their value cannot be changed.

Syntax

val variableName: Type = value

Example

val pi: Double = 3.14

Mutable Variables (var)

Mutable variables are declared using the var keyword. Their value can be changed after initialization.

Syntax

var variableName: Type = value

Example

var count: Int = 0
count = count + 1 // Allowed

Type Inference

Scala has a powerful type inference system. You can often omit the type, and the compiler will infer it for you.

val name = "John" // Compiler infers type String
var age = 30 // Compiler infers type Int

Common Data Types

Here are some common data types you'll encounter:

  1. Int: 32-bit signed integer
  1. Double: 64-bit floating-point number
  1. String: Sequence of characters
  1. Boolean: true or false
  1. Unit: Represents no value, similar to void in other languages
  1. Any, AnyRef, AnyVal: Root types in Scala's type hierarchy

Unusual Scenarios

  1. Reassigning val: Attempting to reassign a val will result in a compilation error.
  1. Type Mismatch: Assigning a value of the wrong type to a variable will also lead to a compilation error.

Real-World Example

In real-world applications, it's good practice to use val (immutable variables) whenever possible to make the code more predictable and easier to maintain.

val taxRate: Double = 0.05
val price: Double = 100.0
val totalCost: Double = price + (price * taxRate)

Conclusion

Understanding variables and types is fundamental to programming in Scala. Emphasizing immutability (val) and leveraging type inference can lead to cleaner and more maintainable code. As you progress, you'll encounter more complex types like collections, case classes, and custom types, which build on these basic principles.
